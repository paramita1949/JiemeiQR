# 轻量架构约束（性能教训固化）

更新时间：2026-04-26

## 目标
本项目是轻量仓储工具，优先保证：
1. 首屏快开
2. 交互稳定
3. 代码简单可维护

## 强制约束（后续开发必须遵守）

1. 禁止页面级定时轮询刷新（如每秒 `Timer.periodic`）。
   - 刷新只能由用户动作触发：进入页面、返回页面、保存后、下拉刷新。

2. 启动流程不加门控页面和人为延迟。
   - 保持 `runApp -> MaterialApp -> HomeScreen` 最短路径。
   - 启动阶段不叠加复杂状态机。

3. 启动日志默认关闭。
   - 调试埋点仅临时启用，问题定位完成后必须移除或默认关闭。

4. 不引入全局“事件总线式”刷新链路来替代简单页面刷新。
   - 本项目数据量小，优先用明确页面生命周期刷新。

5. 任何“优化”必须先证明收益。
   - 先测再改；若体感变差或复杂度升高，立即回退。

## Android 构建稳定性

若遇到 `AAPT2 Daemon startup failed`，保留以下兜底配置：

- `android/gradle.properties`
  - `org.gradle.workers.max=1`
  - `android.enableAapt2Daemon=false`

## 变更评审检查清单

- 是否新增了定时自动刷新？
- 是否增加了启动链路中的额外等待？
- 是否新增了全局状态/通知复杂度？
- 删除这些新增逻辑后，功能是否依旧成立？

只要任一项回答为“是且非必要”，该变更应拒绝合并。
