# 洁美 订单信息与浙江仓库存需求总结

更新时间：2026-04-26

## 1. 项目定位

现有 App 不再只是“箱贴二维码”工具，需要扩展为一个本地离线使用的仓库订单与库存辅助工具。

首页功能入口建议：

1. QR箱码
2. 订单信息
3. 出库日历
4. 库存明细
5. 局域网迁移
6. 基础资料

说明：`备份导入` 不再作为首页独立入口，避免和 `局域网迁移` 的发送/接收数据库能力重复。

## 2. 已有参考文件

### 2.1 日统计.xlsm

用途：参考板数计算逻辑。

核心 VBA 逻辑：

- 按产品行计算。
- 读取多个“箱数”列。
- 汇总箱数。
- 使用“规格”计算板数。

计算规则：

```text
汇总箱数 = 所有箱数列相加
板数 = 汇总箱数 / 规格

如果整除：显示 X板
如果有余数：显示 X板+Y箱
如果不足一板：显示 Y箱
```

示例：

```text
1000箱 / 35箱每板 = 28板+20箱
```

## 3. 产品基础资料

产品基础资料由用户手动维护，不做 Excel 导入作为第一版必需功能。

产品字段：

```text
产品代码
产品名称
默认规格：多少箱/板
```

批号字段：

```text
产品ID
日期批号
实际批号
初始入库数量
入库日期
位置
备注
```

说明：

- 用户平时主要看“日期批号”，例如 `2029.8.03`。
- 实际批号例如 `FBDAXEZ` 也要保留。
- 录入订单时优先显示日期批号，同时显示实际批号。

示例：

```text
72067 六神花露水195ML
规格：35箱/板

批号：
- 日期批号：2029.8.03
  实际批号：FBDAXEZ
  初始库存：103560箱
  位置：4楼-后
```

## 4. 库存规则

库存对象是“批号库存”，必要时可扩展到“批号 + 位置库存”。

当前业务规则：

- 浙江仓是当前库存主体。
- 移库是从浙江仓移到别的仓库，对浙江仓来说是库存减少。
- 移库只需要手动备注，不需要维护目标仓库字段。
- 订单未完成时不扣库存。
- 运单标记完成时才扣库存。
- 库存为 0 的批号在库存页面标红。
- 库存为 0 的批号在订单选择时不再出现。

库存减少类型：

```text
订单出库
移库出库
损耗调整
```

库存增加类型：

```text
初始入库
入库调整
```

动态库存计算：

```text
某日期库存 =
初始入库数量
+ 截止该日期的入库调整
- 截止该日期的订单出库
- 截止该日期的移库出库
- 截止该日期的损耗调整
```

选择历史日期时，必须显示该日期当时的库存快照。

示例：

```text
选择 4月21日：
显示 4月21日结束时总库存
显示每个产品在 4月21日结束时库存
显示每个批号在 4月21日结束时库存
```

## 5. 订单信息功能

订单按日期管理。

需要同时支持：

- 单个运单
- 每日总订单
- 未完成订单
- 已完成订单

订单字段：

```text
日期
运单号
商家简称
状态：未完成 / 已拣货 / 完成
备注
```

订单明细字段：

```text
订单ID
产品ID
批号ID
箱数
规格
位置
板数结果
```

订单明细规则：

- 一个运单可以有多个产品。
- 同一个产品可以跨多个批号。
- 明细最小单位是“产品 + 批号 + 位置 + 箱数”。
- 订单录入时只能从基础资料选择产品和批号。
- 不能手动输入不存在的批号作为正式明细。
- 选择批号时只显示所选日期库存大于 0 的批号。
- 输入箱数不能超过该日期可用库存。

### 5.1 新增运单页面交互规范（2026-04-26确认）

商家选择：

- 商家字段使用选择控件，不在表单内展开商家卡片。
- 打开商家选择控件后，展示历史商家 Top10（按最近使用频率排序）。
- 支持手动输入新商家。

产品明细首行：

- 明细首行使用标题头：`产品`、`批号`。
- 标题下方展示当前选择值：`产品 72067`、`批号 FCHBLEZ 2029.9.7`。
- 日期批号格式统一使用：`YYYY.M.D`（示例：`2029.9.7`）。
- 产品值显示产品编号，不显示产品长名称。
- 批号默认调用该产品最近使用批号。

箱数与换算显示：

- 输入箱数后，显示换算结果，文案使用：`需配 X板+Y箱`（不用“自动”）。
- 同时显示可用库存：`可用 N箱`。
- 规格显示为：`40箱/板  30件/箱`（示例格式，按产品规格动态替换）。

底部信息：

- 底部仅显示：`已录入 X 条`。
- 不显示“合计”文案。

### 5.2 库存明细页面交互规范（2026-04-26确认）

页面定位：

- 该页面用于查看批号库存明细、筛选、编辑备注。
- 页面名称使用：`库存明细`。

入口分工：

- 首页 `库存明细` 入口进入本页面。
- 首页 `基础资料` 入口进入基础信息录入页面。
- 两者职责分离，不混用。

明细展示字段：

- 每条批号明细至少展示：
  - 产品编号
  - 批号（日期批号格式 `YYYY.M.D` + 实际批号）
  - 库存（件/箱）
  - 板数（`X板+Y箱`）
  - 是否发过
  - 备注

备注编辑：

- 备注支持在 `库存明细` 页面直接编辑（高频操作）。
- 不要求用户跳转到基础信息录入页才能修改备注。

删除操作位置：

- 删除基础资料属于低频高风险操作，不放在库存明细页。
- 删除按钮放在 `基础信息录入` 页，且需二次确认。

库存明细页底部操作：

- 仅保留一个 `录入 / 编辑资料` 入口按钮。
- 不保留“批号资料维护”说明块和删除按钮。

示例：

```text
运单号：168888888
商家：A店

72067 六神花露水195ML
- 2029.8.03 / FBDAXEZ / 500箱 = 14板+10箱
- 2029.8.24 / FBYBBEZ / 300箱 = 8板+20箱

20380 六神180ML止痒花露水喷雾
- 2029.6.12 / ELMAXEZ / 200箱 = 4板
```

## 6. 状态与扣库存

订单状态：

```text
未完成
已拣货
已完成
```

状态规则：

- 新订单默认未完成。
- 未完成订单可以修改明细，不扣库存。
- 已拣货订单表示仓库已处理但未最终完成，不扣库存。
- 点击完成后弹出确认，确认后生成订单出库流水。
- 完成后扣减对应批号库存。
- 完成订单如需修改，需要考虑是否反向恢复库存。

第一版建议：

```text
已完成订单允许撤回为未完成
撤回时删除或反冲对应订单出库流水
```

## 7. 出库日历

日历用于按日期查看库存和业务统计。

不单独设置 `每日订单` 页面。用户在出库日历选择日期或范围后，点击 `查看订单信息` 进入 `订单信息` 页面，并带上当前日期/范围筛选条件。

日期页面需要显示：

```text
总库存件数
订单出库箱数
移库出库箱数
损耗调整箱数
完成订单数
未完成订单数
批号库存列表
```

总库存口径：

- 总库存只显示件数，不显示总箱数。
- 箱数、板数只在批号、产品、订单明细等局部对象中展示。

批号库存显示规则：

- 库存大于 0：正常显示。
- 库存等于 0：红色显示，并标注“已归零”。

示例：

```text
2026-04-26

总库存：3,317,144件
订单出库：12,000箱
移库出库：5,000箱
损耗调整：18箱

72067 六神花露水195ML
- 2029.9.06 / FCGBKEZ / 55,740箱 / 1592板 / 位置：4楼-后一右
- 2029.9.26 / FC2CBEZ / 0箱 / 已归零
```

## 8. 移库出库

移库对浙江仓来说是库存减少。

移库记录字段：

```text
日期
产品
批号
位置
数量
备注
```

不做目标仓库结构化字段，目标仓库或原因写入备注。

示例：

```text
类型：移库出库
数量：500箱
备注：移到上海仓
```

## 9. 数据库建议

推荐：

```text
SQLite + drift
```

原因：

- Flutter 本地离线使用合适。
- 适合订单、产品、批号、库存流水这种关系型数据。
- 支持按日期统计、筛选、排序、汇总。
- 迁移和备份简单。
- 后续字段变化可做数据库迁移。

不建议第一版使用云数据库。

## 10. 备份、导入、迁移

需要支持：

- 完整备份导出。
- 完整备份导入。
- 表格导出，用于查看、对账、打印。
- 局域网配对迁移。

完整备份：

```text
jiemei_backup_YYYY-MM-DD.jmq
```

备份内容：

```text
SQLite 数据库文件
backup_info.json
```

局域网迁移：

```text
旧手机：发送数据
新手机：接收数据
同一 Wi-Fi 下扫码或输入配对码
传输完整备份包
```

建议实现：

```text
发送端启动临时局域网 HTTP 服务
二维码包含 IP、端口、一次性验证码
接收端扫码下载备份包
导入前自动备份当前数据
传输完成后关闭服务
```

## 11. 待确认问题

1. 同产品跨批号时，整单板数合计是按每个批号分别算，还是同产品合并后再算。
2. 已完成订单是否允许修改；如果允许，库存流水如何反冲。
3. 位置是否可能同一批号拆分多个位置。
4. 是否需要商家简称基础库。
5. 表格导出第一版需要导出哪些表：订单、库存快照、出库日历、库存流水。
