# Changelog

## 3.2.3 (2026-04-30)

### Added
- 订单信息新增“异常”快捷标签，用于集中查看因基础资料批号修改触发的历史异常订单。
- 订单明细新增异常标记字段，数据库升级到 schema 5，覆盖安装后会自动迁移并新增字段。
- 运单详情产品明细新增“异常”标记，便于逐条人工修正批号。
- 新增异常订单筛选、已完成异常订单修正、数据库迁移与 OCR 日期匹配测试覆盖。

### Changed
- 基础资料修改批号或日期后，若同产品同日期存在多个实际批号，会自动把相关历史订单明细归入异常，包括已完成订单。
- 异常订单不自动猜测正确批号，需人工进入订单详情逐条修正。
- 已完成订单允许修正异常明细，并同步调整对应出库流水，保持库存一致。
- OCR 批号匹配优先按截止日期识别，支持 `2029.08.11` 与 `2029.8.11` 等日期格式互通；同日期多批号时仍需实际批号区分。

## 3.2.2 (2026-04-30)

### Added
- 新增运单页接入“拍照识别”入口，支持拍照/相册图片调用 Gemini OCR 识别发货单模板字段。
- 新增拍照识别结果确认页，识别后先匹配本地产品与批号，再由用户确认录入。
- 首页新增“AI配置”，可在 App 内填写 Gemini API Key 与模型名称，默认模型为 `gemini-3-flash-preview`。
- 新增 Gemini OCR 服务、OCR 行匹配、重复批号合并、AI配置存储与确认录入的测试覆盖。

### Changed
- Gemini 识别仅作为 OCR/模板字段抽取使用，不参与库存、批号归属或业务判断；库存校验、重复明细处理和保存仍复用 App 内部逻辑。
- OCR 明细仅合并同产品、同实际批号的行，不同实际批号保持分开录入。
- 运单号调整为全局唯一业务标识；新增、OCR录入、详情编辑和底层订单创建遇到重复运单号时提示“运单号已存在”。
- 修复热重载后新增运单页点击“拍照识别”可能因图片选择器未初始化而崩溃的问题。

## 3.2.1 (2026-04-30)

### Changed
- 局域网互传发现协议新增多端点候选地址，发送端会广播多个可用局域网地址，避免 Wi-Fi、移动数据、VPN 同时开启时只选到错误 IP。
- 接收端发现设备后新增 `/hello` 主动探测与身份校验，会并发验证候选地址并使用最快可达的匹配端点。
- 附近设备列表按设备与会话去重，减少同一设备因多网卡重复出现。
- 数据备份页面继续简化操作文案与布局，保留“生成备份 / 导入备份 / 清理备份 / 重置数据”等核心动作。

### Added
- 新增多端点发现协议、错误端点兜底连接、备份分享导入、精简备份界面的测试覆盖。

## 3.2.0 (2026-04-30)

### Added
- 手机端数据备份新增“生成并分享”，可将当前数据库快照打包为 `.jiemei` 文件并调起系统分享面板，支持通过微信、QQ、文件助手、网盘等工具发送。
- 备份快照列表新增单条“分享”，可直接分享历史快照。
- 新增“导入备份文件”，支持从系统文件选择器导入 `.jiemei` 分享包或 `.sqlite` 快照，导入前会自动备份当前数据。

### Changed
- 备份分享包内包含 `manifest.json` 和 sqlite 快照，方便后续版本校验和跨设备恢复。

## 3.1.9 (2026-04-29)

### Changed
- 手机端局域网传输改为“附近设备自动发现”为主要接收流程，接收端会先搜索可发送设备并选择连接。
- 发送端收到接收请求后必须手动确认，确认后才下发一次性传输令牌，避免同一 Wi-Fi 内误接收数据。
- 保留二维码与 6 位配对码作为兜底方案，适配移动数据、VPN、多网卡或 UDP 广播不可达的网络环境。
- 增强拒绝、超时、网络异常等状态提示，接收端能明确看到发送端未确认、已拒绝或连接失败。

### Added
- 新增主动发现探测与设备公告响应，提升接收端发现附近发送设备的成功率。
- 新增局域网发现、传输请求、确认令牌、拒绝/超时路径的测试覆盖。
