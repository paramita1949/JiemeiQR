# Changelog

## 4.6.0 (2026-07-09)

### Changed
- 交货计划飞桨 OCR 新增单元格文本表格解析，在无 Markdown/HTML 表格结果时也能按表头和列宽还原明细行。
- 交货计划 OCR 表头识别增强，支持产品码、产品名称、批号、日期批号和“交货计划”筛选列等更多列名变体。
- 交货计划 OCR 会用“交货计划/Σ交货计划”列过滤 0 数量行，并继续优先读取“减交货计划可用量箱数”列。
- 交货计划复核和详情明细将编号、批号、日期与箱数、板数分组换行显示，减少窄屏横向滚动。
- 交货计划飞桨 OCR 进度提示改为更简洁的排队、识别中和已处理进度文案。
- 应用代码版本号更新为 4.6.0。

## 4.5.9 (2026-07-09)

### Changed
- 交货计划 OCR 新增“交货计划/Σ交货计划”箱数字段，只保留交货计划为正数的明细行。
- 交货计划会结合基础资料的每箱件数修正“减交货计划可用量箱数”误读为件数的问题，减少预备箱数偏差。
- 运单 OCR 的商家识别接入历史商家匹配，支持行政区划、公司后缀和短名称归一，提高商家简称回填准确率。
- 运单号识别规则调整为保留清晰可见的异常位数号码并提示复核，不再直接清空非 7 位候选。
- 订单编辑页历史商家选择入口改为商家输入框右侧按钮，飞桨 OCR 识别进度文案更简洁。
- 应用代码版本号更新为 4.5.9。

## 4.5.8 (2026-07-08)

### Changed
- 交货计划明细行调整为第一行横向显示编号、批号、日期、预备箱数和板数，第二行单独显示库位。
- 交货计划基础资料匹配增强：可在 OCR 缺少物料号时通过批号和日期唯一匹配本地产品、库位、每板箱数和 APP 可用箱数。
- 交货计划 OCR 上传会按图片内容自动使用 PNG、JPEG 或 WebP MIME 类型，减少截图格式不一致导致的识别异常。
- 交货计划 OCR 提示词补充逐行对齐和漏行检查规则，减少同物料不同批次被漏识别或跨行拼接。
- 应用代码版本号更新为 4.5.8。

## 4.5.7 (2026-07-08)

### Changed
- 交货计划复核和详情明细改为统一列表样式，突出产品批号、日期、库位和预备箱数。
- 交货计划明细新增每板箱数匹配，预备箱数会同步显示对应板数，便于现场备货。
- 交货计划 OCR 提示词改为先按“交货计划”正数筛选行，再读取“减交货计划可用量箱数”，减少无关物料进入复核。
- 自动签到前台提醒不再弹出确认框，改为只使用通知文案，降低操作打断。
- 应用代码版本号更新为 4.5.7。

## 4.5.6 (2026-07-07)

### Changed

- 自动围栏签到成功后会将自定义文案带入系统通知，前台打开 APP 时同步弹出确认提示。
- 考勤规则数据表新增自动签到弹窗字段，数据库 schema 升级到 23，并在备份 schema 版本中同步记录。
- 应用代码版本号更新为 4.5.6。

## 4.5.5 (2026-07-07)

### Changed
- 交货计划入口和空状态图标改为日程便签样式，更贴合交货计划场景。
- 交货计划记录时间统一显示为“月.日 时:分”，减少“今天”和年份显示造成的视觉干扰。
- Gemini 与魔搭运单 OCR 提示词新增运单号校验，只保留数字字符并要求去前导 0 后为 7 位，无法确认时返回空值并提示疑似识别错误。
- 应用代码版本号更新为 4.5.5。

## 4.5.4 (2026-07-07)

### Changed
- 交货计划明细新增库位字段，数据库 schema 升级到 22，并在备份 schema 版本中同步记录。
- 交货计划识别后会根据产品码、批号和截止日期匹配本地基础资料，自动补齐库位并使用 APP 可用箱数计算预备箱数。
- 交货计划 OCR 提示词调整为只识别“减交货计划可用量箱数”，不再让 AI 提取或推算在库总箱数、库位等本地字段。
- 交货计划复核和详情卡片改为紧凑显示产品码、批号、日期、库位和预备箱数，减少无关库存字段干扰。
- 应用代码版本号更新为 4.5.4。

## 4.5.3 (2026-07-06)

### Changed
- 交货计划 AI 扫描弹窗新增谷歌、魔搭、飞桨 OCR 通道切换，可在识别前选择当前使用的 OCR 服务。
- 交货计划 AI 扫描弹窗支持切换对应通道的模型预设，并显示简短模型名称。
- 交货计划 AI 扫描会保存本次通道和模型选择到 AI 配置，后续识别沿用最新选择。
- 未配置凭证的 OCR 通道会在弹窗中置灰，减少误选后识别失败。
- 应用代码版本号更新为 4.5.3。

## 4.5.2 (2026-07-06)

### Changed
- 交货计划 AI 扫描新增飞桨 OCR 通道，按 AI 设置中的飞桨配置提交识别任务。
- 飞桨交货计划识别支持异步提交、轮询、下载结果和进度提示，识别过程会记录调试事件。
- 交货计划飞桨结果解析支持 Markdown 与 HTML 表格，自动提取物料号、批次、截止日期、在库总箱数和减交货计划可用量。
- 飞桨交货计划 OCR 增加提交、查询和下载阶段的超时与网络异常提示，减少长时间无反馈。
- 应用代码版本号更新为 4.5.2。

## 4.5.1 (2026-07-06)

### Changed

## 4.4.9 (2026-06-25)

### Changed
- 运单 OCR 明细会按“产品码+批号+截止日期”聚合相同行，自动累加箱数，减少同批次被拆成多行的问题。
- Gemini 与魔搭 OCR 提示词同步要求同批次明细聚合，不同批号或不同截止日期仍保持分开。
- 飞桨 OCR 改用统一明细聚合逻辑，保留总箱数校验，避免简单去重导致箱数丢失。

## 4.4.8 (2026-06-25)

### Changed
- 运单 OCR 新增图片总箱数识别，支持将明细箱数合计与页小计/总计进行校验并提示不一致。
- 飞桨 OCR 解析 HTML 表格时会读取页小计和总计箱数，优先使用总计结果辅助校验。
- Gemini 与魔搭 OCR 提示词补充总箱数字段和合计校验要求，减少漏行或箱数识别错误不易发现的问题。
- OCR 批号匹配在产品码已识别但日期批号缺失时，会按可用库存和最近日期代选批号并标记需复核。
